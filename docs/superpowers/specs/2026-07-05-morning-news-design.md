# morning-news 設計書

作成日: 2026-07-05
ステータス: 承認済み

## 目的

毎朝、日本国内・世界のニュースの見出しを1ページで確認できるツール。
スマホのブラウザから URL を開くだけで閲覧できる。

## 全体アーキテクチャ

- リポジトリ: GitHub 公開リポジトリ `morning-news`
- 実行基盤: GitHub Actions（cron スケジュール、毎朝 6:00 JST = 21:00 UTC 前日）
- 生成: Python スクリプト（標準ライブラリのみ、Python 3.9 互換）が各ソースの RSS を取得し、単一の静的 `index.html` を生成
- 公開: GitHub Pages。URL は `https://<ユーザー名>.github.io/morning-news/`
- スマホ側は静的ページを読み込むだけ。JS によるフィード取得は行わない（CORS 回避・高速表示のため）

## コンポーネント

### 1. `build.py`（生成スクリプト）

- 役割: ソース定義に従い RSS を取得 → パース → `index.html` を出力
- 入力: スクリプト内のソース定義リスト（セクション名・ソース名・RSS URL の組）
- 出力: リポジトリ直下の `index.html`（Pages は main ブランチのルートから配信）
- 依存: Python 標準ライブラリのみ（`urllib.request`, `xml.etree.ElementTree`, `email.utils` など）。pip install 不要
- ソースの追加・削除は定義リストに1行足す/消すだけで完結する

### 2. ソース定義（初期構成）

| セクション | ソース（RSS） |
|---|---|
| 国内 総合 | NHKニュース 主要 |
| 世界 | NHK国際、BBC News Japan、ロイター（Reuters World） |
| 経済・ビジネス | NHK経済、Yahoo!ニュース 経済 |
| テクノロジー・IT | ITmedia NEWS、はてなブックマーク テクノロジー人気 |
| スポーツ・エンタメ | Yahoo!ニュース スポーツ、Yahoo!ニュース エンタメ |

- 各セクションは新しい順に最大10件表示
- 実装時に各 RSS URL の生存確認を行い、死んでいるものは同等の代替に差し替える

### 3. `index.html`（生成物）

- 1カラム縦並び、スマホ最適化（viewport 設定、相対単位）
- 上部にジャンルタブ（ページ内アンカーへジャンプ）
- 各記事: 見出し（タップで元記事へ新規タブで遷移）＋ソース名＋「○分前/○時間前」の相対時刻
- ダークモード自動対応（`prefers-color-scheme`）
- ページ最上部に最終更新時刻（JST）を表示
- CSS はインライン埋め込み。外部リソース読み込みなし

### 4. GitHub Actions ワークフロー

- トリガー: `schedule`（毎朝 6:00 JST）＋ `workflow_dispatch`（手動実行）
- 処理: checkout → `python build.py` → 生成された `index.html` に差分があれば commit & push
- Pages は該当ブランチ/ディレクトリから自動配信

## エラー処理

- 個別ソースの取得失敗: 該当ソースをスキップして他セクションの生成を継続。失敗したソースはページ内に「取得失敗」と小さく表示
- 全体失敗（Actions 自体の失敗）: 前回生成のページがそのまま残る。ページが消えることはない
- RSS の日付形式ゆれ: RFC822 / ISO8601 の両対応。パース不能なら時刻表示を省略して見出しのみ表示

## テスト方針

- `build.py` はフィード XML のパース部分を関数分離し、ローカルの固定 XML フィクスチャでユニットテスト（pytest 不使用でも `unittest` で可）
- ネットワーク取得部は実行時の手動確認（`python build.py` をローカル実行して index.html を目視）
- Actions は `workflow_dispatch` で手動実行して疎通確認

## 制約・前提

- Mac ローカルの Python は 3.9.6 のみ → スクリプトは 3.9 互換で書く（Actions 側も 3.9 を指定するか互換コードにする）
- 公開リポジトリのためページは誰でも閲覧可能（ニュースリンク集のため問題なしと確認済み）
- 費用: すべて無料枠内（GitHub Actions 無料枠、Pages 無料）
- ユーザーの GitHub アカウントは既存。gh CLI は未インストールのためセットアップ時に導入案内する

## スコープ外（YAGNI）

- 通知（LINE/メール等）
- 既読管理・ブックマーク
- 検索機能
